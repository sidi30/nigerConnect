NigerConnect — Architecture & Flux

Réseau social pour la diaspora nigérienne (62 000+ membres, 15+ pays). Monolithe modulaire NestJS côté serveur, app mobile React Native Expo, site vitrine Next.js 15. Une seule base PostgreSQL, cache Redis, stockage objet S3/MinIO, temps réel Socket.io.

1. Vue d'ensemble

Trois applications communiquent avec une seule API REST + WebSocket. L'API parle à Postgres (données), Redis (cache, présence, file BullMQ), et S3/MinIO (fichiers). Les notifications sortantes passent par FCM, Resend et Twilio.

HTTPS / WS client → API API → infra (Prisma, Redis, S3) Jobs asynchrones (BullMQ)

3. Les 12 modules de l'API

Chaque module est autonome et ne dépend que de common/ + Prisma. La communication inter-modules se fait via l'injection de dépendances NestJS — jamais d'import direct d'un autre module.

authRegister/Login, OAuth Google/FB/Apple, JWT RS256 (15 min) + refresh rotatif (30 j), vérif identité, détection vol de token.
profileProfil, bio, photos (presign S3), recherche par ville/pays/langue.
socialAmis, demandes, suggestions, blocages.
feedPosts, stories 24h, likes, commentaires, partage.
chatREST pour conversations/messages + Gateway Socket.io /chat + présence Redis.
geoSnap Map — membres proches, clustering par zoom (PostGIS).
marketplaceServices d'entraide, réponses, ratings.
associationAssociations, événements, membres, join/leave.
notificationPush FCM, Email Resend, SMS Twilio, groupage via BullMQ.
moderationSignalements, file de modération, actions admin.
mediaSharp pour les transformations + upload S3.
commonGuards, pipes, décorateurs, config Zod, Prisma, Redis, mail, throttle, filtres d'exception.

4. Flux d'authentification

JWT d'accès court (15 min) + refresh token rotatif (30 j) stocké hashé en base. La rotation détecte la réutilisation d'un refresh token déjà consommé = révocation totale.

Requête client Appel serveur interne Réponse

5. Flux Feed (posts & likes)

Les écritures sont immédiates en DB, les fan-out (notifications) sont délégués à BullMQ pour garder des réponses HTTP rapides.

Mobile → API API → Postgres (Prisma) Jobs asynchrones / cache

6. Flux Chat temps réel

Le client ouvre un WebSocket sur le namespace /chat. La connexion est authentifiée par le même JWT RS256 que REST. La présence en ligne est stockée dans Redis.

WebSocket client → serveur Persistance / présence Delivery (WS ou push)

7. Flux Upload Média (photos profil)

Pour éviter que l'API ne proxy de gros fichiers, elle signe une URL S3/MinIO. Le mobile uploade directement vers l'objet, puis confirme à l'API qui met à jour la DB.

Requêtes mobile API → DB Réponses

8. Flux Notifications (groupées)

Pour éviter le spam, les notifications sont agrégées dans BullMQ : un worker ramasse les events similaires dans une fenêtre (ex. 5 min), puis envoie une seule push / email / SMS.

Événements internes API Queue async (BullMQ/Redis) Appels providers externes

9. Flux Carte / Snap Map

Les coordonnées des membres sont stockées en PostGIS. Le backend agrège les résultats par cluster selon le zoom pour ne pas retourner 60k points bruts.

11. Sécurité — les 12 couches

1

TLS 1.3

Toutes les requêtes chiffrées en transit.

2

Rate limiting

100/min IP · 1000/min utilisateur (AppThrottleModule).

3

JWT RS256

Access 15 min, clé publique vérifiée dans les gateways.

4

Refresh rotatif

Hashé en DB, rotation + détection de vol.

5

Guards NestJS

JwtAuthGuard, RolesGuard, décorateur @Public.

6

Zod strict

strip unknown, types partagés front/back.

7

Helmet

Headers HTTP durcis.

8

CORS whitelist

Origines validées au démarrage (env.validation).

9

argon2id

Hash du password.

10

Docs chiffrés

Identité sur S3 avec AES-256.

11

Anti-SQLi

Prisma uniquement, pas de raw SQL non paramétré.

12

Brute-force lock

5 échecs → 15 min · 30 min · 1 h exponentiel.

13. Démarrage local — en 7 commandes

# 1. Installer
pnpm install

# 2. Env
cp .env.example .env

# 3. Clés JWT RS256
pnpm --filter @nigerconnect/api keys:generate

# 4. Services (Postgres + Redis + MinIO)
docker compose up -d

# 5. Schéma DB
pnpm --filter @nigerconnect/api prisma:generate
pnpm --filter @nigerconnect/api prisma:migrate

# 6. API
pnpm --filter @nigerconnect/api dev          # http://localhost:3000

# 7. Mobile (autre terminal)
pnpm --filter @nigerconnect/mobile dev
